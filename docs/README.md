# Documentacao

Documentacao viva do Linka WebApp.

## Pastas

- `architecture`: estrutura, contratos tecnicos, capacidades web e decisoes arquiteturais.
- `product`: telas, comportamento funcional, componentes, branding e copy.
- `qa`: pendencias, regressao e evidencias PWA.
- `release`: CI/CD, deploy Cloudflare Pages e fluxo Git.
- `ai`: orientacao para agentes, skills e operacao assistida.
- `archive`: historico util que nao e regra atual do WebApp.

## Regras

- Atualize docs na mesma mudanca que altera comportamento, fluxo, contrato ou estrutura.
- Nao usar `archive` como fonte canonica sem validar contra codigo atual.
- Documentacao Android/Capacitor deve ficar fora da documentacao viva deste WebApp.
